# 🥯 Skill: Bun Development

> **PURPOSE:** High-performance runtime development (Bun) for fast execution, bundling, and testing.
> **CATEGORY:** Backend
> **TIER:** 2+

---

## 🚦 ACTIVATION SIGNALS

- Keywords: `Bun`, `runtime performance`, `fast bundling`.
- Task: Setting up new projects with Bun, migrating from Node, or using Bun tools (test, shell, sqlite).

---

## 🛠️ SUB-STEPS (EXECUTION PIPELINE)

1. **Initialization**: Create project with `bun init` or migrate by installing with `bun install`.
2. **Runtime Execution**: Run TS/JS files directly without transpiling (`bun file.ts`).
3. **Bundling**: Use `bun build` for high-speed JS/TS bundling.
4. **Testing**: Implement fast unit/integration tests with `bun:test`.
5. **Standard Library Usage**: Leverage `Bun.serve` for fast HTTP servers and `Bun.sqlite` for speed.

---

## 📝 OUTPUT SIGNATURE

- High-performance servers (TS/JS).
- Fast CI/CD configs for Bun.
- Benchmark comparisons.
