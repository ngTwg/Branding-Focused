# 📖 Skill: Documentation Templates

> **PURPOSE:** Grouped templates and structure guidelines for README, API docs, code comments, and AI-friendly documentation.
> **CATEGORY:** Workflows
> **TIER:** 1+

---

## 🚦 ACTIVATION SIGNALS

- Keywords: `document this`, `create README`, `API docs`, `docstrings`.
- Goal: Systematizing documentation across a project.

---

## 🛠️ SUB-STEPS (EXECUTION PIPELINE)

1. **Information Extraction**: Scan core functionality, entry points, and CLI commands.
2. **Template Selection**: Choose the right header set for README, API, or Runbooks.
3. **Execution**: Populating the template with project-specific details.
4. **Consistency Pass**: Ensure tone is consistent with technical/business goals.

---

## 📝 OUTPUT SIGNATURE

- Complete Documentation artifacts (README.md, docs/api.md, etc.).
- JSDoc/TSDoc blocks if relevant.
- High-level overview of project structure.

---

## 🧪 BENCHMARK TASK

- **Input**: "Create a professional README for this new library."
- **Output**: README with: Title -> Quick Start -> Features -> Configuration -> API Reference.
