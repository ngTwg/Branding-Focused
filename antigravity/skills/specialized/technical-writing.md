# ✍️ Skill: Technical Writing

> **PURPOSE:** Convert raw information or code into high-quality documentation.
> **CATEGORY:** Specialized
> **TIER:** 1+

---

## 🚦 ACTIVATION SIGNALS
- Keywords: `document`, `write-up`, `explain`, `technical report`, `blog post`, `README`.
- Task: Finalizing a project, creating onboarding docs, or explaining complex concepts.

---

## 🛠️ SUB-STEPS (EXECUTION PIPELINE)

1. **Audience Profiling**: Identify if the reader is a "Newbie", "Developer", "Exec", or "Expert".
2. **Structural Design**: Choose the right template (Tutorial vs Reference vs Guide).
3. **Drafting**: Write clear, active-voice prose, avoiding "actually", "just", "simply", and "obviously".
4. **Visual Integration**: Embed code blocks (with highlighting), diagrams (Mermaid), and tables.
5. **Clarity Review**: Simplify long sentences and use 2nd person ("You") for instructions.

---

## 📝 OUTPUT SIGNATURE
- Polished Markdown document.
- Zero passive voice (target).
- 100% correct code snippets in context.

---

## 🧪 BENCHMARK TASK
- **Input**: "Write a developer guide for this new auth module."
- **Output**: Installation -> Usage -> Types -> API Reference -> Troubleshooting.
