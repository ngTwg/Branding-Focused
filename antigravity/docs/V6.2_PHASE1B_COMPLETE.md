
**New FailureEntry Fields:**
```python

times_helped: int = 0       # How many times prevented same error
effectiveness: float = 0.0  # times_helped / times_injected
```

### 3. Retrieval Scoring Upgrade (100%)

**File:** `antigravity/core/failure_memory.py`

**New Scoring Weights:**
- 40% - Effectiveness (times_helped / times_injected) ← HIGHEST
- 30% - Context match (file_type, framework, error_type)
- 20% - Pattern type match
- 10% - Recency + Frequency

**Filter:** Only inject patterns with `confidence_score > 0.6`

### 4. Prompt Injection Upgrade (100%)

**Changed from "advice" to "constraints":**

**Before (v1):**
```
Avoid: - missing import
```

**After (v2):**
```
[FAILURE MEMORY - CRITICAL CONSTRAINTS]
You are making a repair to code that previously failed.

1. ADDED NULL CHECK FOR 'useState is not a function' ERROR [error_type=not_a_function]
   Previous attempt: added null/undefined check before calling useState
   Why it failed: check imports - function likely not imported or imported incorrectly
   Confidence: 80% (seen 8 times) | Relevance: 85%

STRICT RULES:
  1. Adding null checks when error is 'X is not a function' - root cause is usually wrong import
     → MUST: Check import statements first: verify function is imported, check for typos, verify package version

If you violate these rules, the patch will likely fail again.
```

### 5. Effectiveness Tracking (100%)

**File:** `antigravity/scripts/orchestrator.py`

**Integration Points:**

1. **Pattern Injection (replan_repair):**
   ```python
   injected_pattern_signatures = []
   for pattern, lesson, score in relevant_failures:
       injected_pattern_signatures.append(pattern.signature)
       self.failure_memory._store._increment_injection_count(pattern.signature)
   ```

2. **Success Tracking (execute loop):**
   ```python
   if error_delta.net_improvement and error_delta.new_error_score == 0.0:
       for sig in injected_pattern_signatures:
           self.failure_memory._store._increment_helped_count(sig)
   ```

3. **Effectiveness Computation:**
   ```python
   effectiveness = times_helped / times_injected
   ```

**Helper Methods Added:**
- `_increment_injection_count(signature)` - Track when pattern shown to LLM
- `_increment_helped_count(signature)` - Track when pattern prevented error

## 🎓 KEY LEARNINGS

### What Worked:
1. **Signal pairing** - Combining error + patch + context creates diagnostic insights
2. **Effectiveness tracking** - Measuring real impact drives quality
3. **Constraint-style prompts** - Directive language changes LLM behavior
4. **Context filtering** - confidence_score > 0.6 reduces noise

### Critical Insights:
1. **Quality > Quantity** - 10 high-signal patterns > 100 generic ones
2. **Measurement is key** - Can't improve what you don't measure
3. **Context is everything** - Generic patterns don't change behavior
4. **Specificity wins** - "Check imports" > "Fix the error"
