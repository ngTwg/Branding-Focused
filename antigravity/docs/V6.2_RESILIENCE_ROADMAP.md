## 🎯 Executive Summary

Based on production readiness review, v6.1 achieved **production-grade** (correct, tested, observable) but lacks **production resilience** for real workloads. V6.2 addresses 5 critical gaps:

1. ✅ **Learning Loop** - Patch failure memory (HIGHEST IMPACT)
2. ✅ **Index Lifecycle** - Maintain retrieval quality
3. ✅ **Budget Strategy** - Adaptive behavior when low budget
4. ✅ **Error Prioritization** - Focus on root causes
5. ✅ **Health Monitoring** - Self-evaluation with actionable metrics

---

## 📊 Gap Analysis

### Gap #1: HybridRetriever - No Index Lifecycle
**Problem:** Skills thay đổi → embeddings stale → retrieval quality degrades (silent failure)

**Solution:** IndexManager
- Filesystem watch / checksum comparison
- Incremental re-indexing
- Stale ratio tracking
- Auto-trigger reindex at 20% threshold

**Impact:** Prevent silent quality degradation

---

### Gap #2: ASTAnalyzer - Multi-Error Overload
**Problem:** Nhiều errors → LLM context overload → fix sai root cause

**Solution:** ErrorPrioritizer
- Sort by severity (syntax > runtime > lint)
- Detect error chains (import missing → undefined name)
- Cluster similar errors
- Limit context ≤ 1000 tokens

**Impact:** Focus on root causes, reduce wasted repairs

---

### Gap #3: Delta System - No Learning
**Problem:** LLM lặp lại cùng sai lầm → retry ngu → waste tokens

**Solution:** FailureMemory (HIGHEST IMPACT)
- Record failed patch signatures
- Extract failure patterns
- Similarity search on errors
- Inject memory into repair prompts

**Impact:** 30%+ retry reduction, 20%+ token savings

---

### Gap #4: BudgetGuard - Not Strategy-Aware
**Problem:** Budget thấp → vẫn dùng full features → fail fast

**Solution:** BudgetStrategy
- 3 zones: Green (>50%), Yellow (20-50%), Red (<20%)
- Adaptive behavior per zone
- Yellow: reduce top_k, shorter prompts, prefer SLM
- Red: minimal mode, disable repair loop

**Impact:** Maximize success with limited resources

---

### Gap #5: Tracing - Not Actionable
**Problem:** Dashboard đẹp nhưng không giúp optimize

**Solution:** HealthMonitor
- Derived metrics (rollback_rate, token_per_task, etc.)
- Health score (0-100) with categorization
- Anomaly detection
- Concrete action suggestions

**Impact:** Enable data-driven optimization

## 💡 Design Principles

### 1. **Adaptive over Static**
Don't just enforce limits → adapt behavior based on context

### 2. **Learning over Repeating**
Don't just retry → learn from failures and avoid patterns

### 3. **Proactive over Reactive**
Don't just log errors → detect anomalies and suggest actions

### 4. **Self-Evaluable over Observable**
Don't just collect metrics → compute health score and report

### 5. **Resilient over Robust**
Don't just handle errors → prevent degradation and recover gracefully
