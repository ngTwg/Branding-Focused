### 2. PatternExtractor v2 ✅
**File:** `antigravity/core/pattern_extractor_v2.py` (NEW - 600 lines)

**10 High-Signal Patterns with Signal Pairing:**
1. `fix_symptom_not_root` - Optional chaining for undefined (error + defensive code)
2. `wrong_fix_strategy` - Null check for "not a function" (function error + null check)
3. `wrong_file_modified` - Changed child when error in parent (error file ≠ modified file)
4. `test_breaking_change` - Changed signature without tests (test error + signature change)
5. `incomplete_fix` - Fixed 1 occurrence, missed others (same error + small patch)
6. `overfix` - Too many changes for one error (simple error + large patch)
7. `wrong_assumption` - Assumed array, got object (type error + array method)
8. `dependency_not_updated` - New API without upgrade (import error + new import)
9. `incorrect_refactor_scope` - Refactored too much (multiple files + large changes)
10. `no_op_patch_v2` - Regenerated same code (empty patch)

**Key Innovation: Signal Pairing**
```python
IF:
  error: "is not a function"
  AND patch: added "null check"
THEN:
  pattern = wrong_fix_strategy
  attempted_fix = "added null check"
  correct_direction = "check imports first"
```

   Why it failed: check imports first - function not imported

  1. Adding null checks when error is 'X is not a function' - root cause is usually wrong import

If you violate these rules, the patch will likely fail again.
