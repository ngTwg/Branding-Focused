## 🚨 WEAK SPOT #1: Brittle Pattern Usage Detection

### Current Flaw:
```python
# Keyword matching - easy false positives/negatives
if "import" in patch_diff:
    pattern_used = True
```

**Problems:**
- "import" in comment → false positive
- Fixed import path without "import" keyword → false negative
- Surface-level matching → noisy signal

### Fix: Intent Signature Detection

```python
# In FailurePattern schema, add:
@dataclass
class FailurePattern:
    # ... existing fields ...

    # v4: Intent signatures for precise usage detection
    usage_signals: list[str] | None = None  # What to look for in patch

# In PatternExtractorV2, define signals:
def _rule_wrong_fix_strategy(self, ...):
    pattern = FailurePattern(
        pattern_type="wrong_fix_strategy.react_hook",
        # ... other fields ...
        usage_signals=[
            "added import",
            "modified import",
            "fixed module reference",
            "import.*react",  # regex pattern
        ]
    )

# In FailureMemoryStore, upgrade detection:
def _check_pattern_usage_v2(self, pattern: FailurePattern, patch_diff: str) -> bool:
    """
    v4: Intent signature detection (more precise than keywords).

    Checks for structured signals rather than surface keywords.
    """
    if not pattern.usage_signals:
        # Fallback to v3 keyword detection
        return self._check_pattern_usage(pattern, patch_diff)

    patch_lower = patch_diff.lower()

    # Check each signal
    for signal in pattern.usage_signals:
        # Simple regex support
        if signal.startswith("regex:"):
            import re
            regex_pattern = signal[6:]  # Remove "regex:" prefix
            if re.search(regex_pattern, patch_lower):
                return True
        else:
            # Structured signal detection
            if signal == "added import":
                # Look for "+ import" or "+import" (diff format)
                if "+ import" in patch_lower or "+import" in patch_lower:
                    return True
            elif signal == "modified import":
                # Look for "- import" followed by "+ import"
                if "- import" in patch_lower and "+ import" in patch_lower:
                    return True
            elif signal == "fixed module reference":
                # Look for changes to module paths
                if "from " in patch_lower and ("." in patch_lower or "/" in patch_lower):
                    return True
            elif signal in patch_lower:
                # Direct match
                return True

    return False
```

**Impact:**
- Less noise in effectiveness scores
- Effectiveness becomes decision-grade
- Pattern ranking stabilizes faster

## 🚨 WEAK SPOT #2: Independent Pattern Attribution

### Current Flaw:
```python
# All patterns get full credit
for sig in injected_pattern_signatures:
    if pattern_used:
        increment_helped_count(sig)  # Each gets +1
```

**Problem:**
- Pattern A: "check imports"
- Pattern B: "add null check"
- Both injected, fix succeeds
- Both get full credit → inflates weak patterns

### Fix: Split Attribution Credit

```python
# In FailureMemoryStore:
def _increment_helped_count_split(
    self,
    signature: str,
    credit: float = 1.0  # v4: fractional credit
) -> None:
    """
    v4: Increment with fractional credit to handle co-occurrence.

    When multiple patterns are injected together, split credit
    to avoid over-crediting weak patterns.
    """
    if signature in self._by_signature:
        entry = self._by_signature[signature]
        entry.times_helped += credit  # Fractional increment

        # Recompute effectiveness
        if entry.times_injected > 0:
            entry.effectiveness = entry.times_helped / entry.times_injected

        self._save_to_disk()
        logger.info(
            f"Pattern {signature[:8]} helped (credit={credit:.2f})! "
            f"Effectiveness: {entry.effectiveness:.2%} "
            f"({entry.times_helped:.1f}/{entry.times_injected})"
        )

# In Orchestrator:
if success and injected_pattern_signatures:
    # Count how many patterns were actually used
    patterns_used = []
    for sig in injected_pattern_signatures:
        entry = self.failure_memory._store._by_signature.get(sig)
        if entry:
            pattern = entry.pattern
            if self.failure_memory._store._check_pattern_usage_v2(pattern, patch_diff):
                patterns_used.append(sig)

    # Split credit among used patterns
    if patterns_used:
        credit_per_pattern = 1.0 / len(patterns_used)
        for sig in patterns_used:
            self.failure_memory._store._increment_helped_count_split(
                sig,
                credit=credit_per_pattern
            )
        logger.info(
            f"Split credit {credit_per_pattern:.2f} among {len(patterns_used)} patterns"
        )
```

**Impact:**
- Avoids over-crediting weak patterns
- Ranking stabilizes faster
- True effectiveness emerges

## 🚨 WEAK SPOT #3: Prompt Hook Missing Final Pass

### Current Flaw:
```
BEFORE WRITING THE FIX, YOU MUST:
1. Identify pattern
2. Explain failure
3. State what's different
4. THEN write patch
```

**Problem:** LLM often explains correctly, then drifts during generation

### Fix: Add Final Alignment Pass

```python
def format_for_prompt_v4(relevant_failures):
    """v4: Add final alignment pass"""

    lines = ["[FAILURE MEMORY - CRITICAL CONSTRAINTS]"]
    # ... existing pattern display ...

    # Forced reasoning hook
    lines.append("BEFORE WRITING THE FIX, YOU MUST:")
    lines.append("1. Identify which failure pattern(s) apply to this error")
    lines.append("2. Explain why the previous fix approach failed")
    lines.append("3. State what you will do differently this time")
    lines.append("4. Write the patch")
    lines.append("5. Verify the fix explicitly satisfies the constraints above\n")  # NEW

    # Strict rules
    lines.append("STRICT RULES:")
    for i, (pattern, lesson, score) in enumerate(relevant_failures, 1):
        lines.append(f"  {i}. {lesson.avoid}")
        lines.append(f"     → MUST: {lesson.prefer}")

    lines.append("\nIf you skip the reasoning, drift during generation, or violate these rules,")
    lines.append("the patch will fail again. Final verification is mandatory.")

    return "\n".join(lines)
```

**Impact:**
- Creates final alignment pass
- Reduces drift during generation
- Boosts compliance significantly
