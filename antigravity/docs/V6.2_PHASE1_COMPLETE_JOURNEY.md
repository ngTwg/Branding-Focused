## 📊 EVOLUTION OF KEY METRICS

### Effectiveness Calculation:

**v1 (Phase 1a):**
```python
effectiveness = times_helped / times_injected  # Naive
```
**Problem:** No measurement, just correlation

**v2 (Phase 1b):**
```python
if success:
    times_helped += 1  # Still correlation
effectiveness = times_helped / times_injected
```
**Problem:** False attribution

**v3 (Phase 1c):**
```python
if success AND pattern_used:  # Causal-ish
    times_helped += 1

if times_injected < 5:
    smoothed = 0.1 * frequency  # Cold start
else:
    smoothed = (times_helped + 1) / (times_injected + 2)  # Bayesian
```
**Problem:** Brittle detection, over-crediting

**v4 (Phase 1d):**
```python
patterns_used = [p for p in patterns if intent_signature_detected(p)]
credit = 1.0 / len(patterns_used)  # Split credit

for p in patterns_used:
    times_helped += credit  # Fractional

smoothed = (times_helped + 1) / (times_injected + 2)
```
**Result:** Precise, stable, decision-grade

### v1 (Phase 1a):
```python
# No detection - assume all patterns help
```

### v2 (Phase 1b):
```python
# No detection - still assume
```

### v3 (Phase 1c):
```python
# Keyword matching
if "import" in patch_diff:
    pattern_used = True
```
**Problem:** Brittle, noisy

### v4 (Phase 1d):
```python
# Intent signature detection
pattern.usage_signals = [
    "added import",
    "modified import",
    "regex:import.*react"
]

for signal in usage_signals:
    if detect_structured_signal(signal, patch_diff):
        return True
```
**Result:** Precise, low noise

## 💬 PROMPT EVOLUTION

### v1 (Phase 1a):
```
Avoid: - missing import
```
**Problem:** Generic advice, ignored

### v2 (Phase 1b):
```
STRICT RULES:
  → MUST: Check import statements first
```
**Problem:** Soft constraint, still ignored

### v3 (Phase 1c):
```
BEFORE WRITING THE FIX, YOU MUST:
1. Identify pattern
2. Explain failure
3. State what's different
4. THEN write patch
```
**Problem:** Explains correctly, then drifts

### v4 (Phase 1d):
```
BEFORE WRITING THE FIX, YOU MUST:
1. Identify pattern
2. Explain failure
3. State what's different
4. Write patch
5. Verify fix satisfies constraints  ← NEW

If you skip reasoning, drift during generation,
or violate rules, patch will fail again.
Final verification is mandatory.
```
**Result:** Hard constraint with final alignment

## 🚨 CRITICAL DECISIONS MADE

### Decision 1: Rule-based vs ML for pattern extraction
**Chose:** Rule-based (Phase 1a)
**Why:** Sufficient for 10 patterns, interpretable, fast
**Result:** ✅ Correct choice, can add ML later if needed

### Decision 2: Correlation vs Causation
**Chose:** Causal attribution (Phase 1c)
**Why:** Correlation metrics are misleading
**Result:** ✅ Critical fix, makes metrics trustworthy

### Decision 3: Raw effectiveness vs Smoothed
**Chose:** Bayesian smoothing (Phase 1c)
**Why:** Cold start noise kills ranking
**Result:** ✅ Essential for stability

### Decision 4: Independent vs Split attribution
**Chose:** Split credit (Phase 1d)
**Why:** Co-occurrence inflates weak patterns
**Result:** ✅ Prevents ranking pollution

### Decision 5: Keyword vs Intent signatures
**Chose:** Intent signatures (Phase 1d)
**Why:** Keywords too brittle for production
**Result:** ✅ Precision upgrade
