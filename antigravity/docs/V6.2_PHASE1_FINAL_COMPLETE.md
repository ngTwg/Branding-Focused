**Impact:** Prevents overestimating effectiveness

---

#### Fix #2: Cold Start Noise (BLOCKING) ✅
**Problem:** 40% weight on noisy early data (1/1 = 100%)

**Solution:**
```python
if times_injected < 5:
    # Cold start: low weight
    effective_score = 0.1 * min(1.0, frequency / 10)
else:
    # Bayesian smoothing: (successes + 1) / (attempts + 2)
    smoothed = (times_helped + 1) / (times_injected + 2)
    effective_score = smoothed
```

**Impact:** Prevents early lucky patterns from dominating

---

#### Fix #3: Forced Reasoning Hook (HIGH) ✅
**Problem:** LLM "understands but ignores" constraints

**Solution:**
```
BEFORE WRITING THE FIX, YOU MUST:
1. Identify which failure pattern(s) apply to this error
2. Explain why the previous fix approach failed
3. State what you will do differently this time
4. THEN write the patch

If you skip the reasoning or violate these rules, the patch will fail again.
```

**Impact:** Forces LLM to acknowledge constraints before acting

## 🎓 KEY LEARNINGS

### What Worked:
1. **Signal pairing** - (error + patch + context) creates diagnostic insights
2. **Attribution checking** - Prevents false positives in effectiveness
3. **Bayesian smoothing** - Handles cold start gracefully
4. **Forced reasoning** - Makes LLM acknowledge constraints
5. **Measurement-driven** - Can't improve what you don't measure

### Critical Insights:
1. **Quality > Quantity** - 10 high-signal patterns > 100 generic ones
2. **Attribution matters** - Correlation ≠ Causation
3. **Cold start is real** - Need smoothing to avoid noise
4. **Specificity wins** - "Check imports" > "Fix the error"
5. **Constraints need teeth** - Soft suggestions get ignored
